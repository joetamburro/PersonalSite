Who am I?

It wasn't too long ago that I was aimlessly traveling around the country, hitting things with sticks, taking in the sights.......making music. I have pursued drumming since I was a wee lad. It's all I ever wanted to do until one fateful night comprised of good food, good music, and good friends. This was the night I was enlightened that there was a little code school down the road that I should give some serious thought to. It turns out that little code school does big things.

"The Iron Yard", a good friend tells me. It sounded like a place I was hoping to never see the inside walls. So, I researched. I applied. I was accepted. Upon successfully completing the most grueling code school that exists, these are now some of my favorite walls. Littered with dry-erase murals, surrounding some really great thinkers, and fostering an environment that demands high-level thought, I have transferred my creative passion in music to a new found passion that is web development.

What did I learn you ask?

How to think like a programmer and learn how to learn as I further develop my skills. I love challenging myself and solving complex problems. My life is about creating things that give an end-user great satisfaction whether, it is playing a concert or building an intricate application. I have learned that road-blocks in programming are more like speed bumps. By taking a step back and applying logical thought to a problem the road then becomes nivagable. I have learned a lot in a short time, including such things as:

  • Git and GitHub - version control skills and team workflows
  • HTML5/CSS3 - responsive design, layouts, animations, typography, etc.
  • JavaScript - unit testing via Mocha and Jasmine, types, functions, objects, scope, closures, constructors, prototypes, JSON, etc.
  • jQuery - DOM manipulation, Non-DOM methods, chaining, callbacks and promises, AJAX, and plugins
  • Underscore.js - templating and functional programming principles
  • Backbone.js - understanding of MVC's, Handling JSON payloads, data persistence, and smart view management
  • Parse - as a backend, along with wrangling of API's
  • Yeoman - front-end package managers, tools, asset loaders, and generators
